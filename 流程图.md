## 1. 模块地图（每个文件负责什么）

```
CNKIBug/
├── run.py              ← 总入口。开机界面、问你要啥（关键词/页数/保存方式），然后喊 scraper 干活
└── cnkibug/
    ├── errors.py       ← 弹错误框（Windows 原生弹窗），只依赖标准库
    ├── ui.py           ← 全程序共用的那一个"控制台"对象，加几个好看的提示横幅
    ├── environment.py  ← 开工前体检：有没有 Edge、桌面在哪
    ├── estimate.py     ← 纯算数：估算这次大概要跑多久
    ├── window.py       ← Windows 专用：把浏览器窗口拽到最前（出验证码时提醒你）
    ├── exporter.py     ← 把抓到的标题写成 .xlsx，三种存法
    └── scraper.py      ← 核心：开浏览器、搜索、翻页、抓标题、处理验证码和中断
```

**记忆口诀**：`run` 问你要啥 → `environment` 体检 → `scraper` 干活（中途靠 `window`/`ui` 提醒、`errors` 报错）→ `exporter` 存盘。`estimate` 只是旁边报个时。

## 2. 整体流程图

```
        你双击程序
            │
            ▼
   ┌─────────────────┐
   │     run.py      │  打印欢迎语
   │   main() 循环   │  check_env() 体检：没 Edge 就弹窗退出
   └────────┬────────┘
            │  问你：单/多关键词？页数？保存方式？
            ▼
   ┌─────────────────────────────────────────────┐
   │   scraper.py  →  scrape_cnki(...)             │
   │                                               │
   │   1) 启动 Edge（失败就退而求其次用 Chromium） │
   │   2) 开一个浏览器标签页 page                   │
   │   3) 预热：先空跑一次搜索，消化首次重定向      │
   │   4) 对每个关键词：_scrape_keyword(page, ...)  │
   └────────────────────┬──────────────────────────┘
                        │（下面是单个关键词的内部）
                        ▼
   ┌─────────────────────────────────────────────┐
   │   _scrape_keyword(page, keyword, max_pages)   │
   │                                               │
   │   打开知网 → 填关键词 → 点搜索                 │

   │   single / multi_split / multi_merge          │

                        │
                        ▼
                  桌面出现 .xlsx
```

## 3. 从头到尾：一次抓取发生了什么（带代码）

### 第 1 步：入口与体检 —— `run.py`

入口先 `import` 那些第三方库（playwright/openpyxl/rich）。如果用户拿到的是坏掉的 exe，这里会 `import` 失败，于是弹个友好窗口而不是报一堆红字：

```python
from cnkibug.errors import _popup_error      # 这个只用标准库，一定 import 得了
try:
    from cnkibug.scraper import scrape_cnki   # 这些会拉起第三方库
    ...
except ImportError as _err:
    _popup_error([... "缺失模块: " + str(_err) ...])   # 弹窗提示
    sys.exit(1)
```

然后 `check_env()` 体检（在 `environment.py`）：Windows 上没装 Edge 就弹窗退出。

接着 `main()` 里一个大 `while True` 循环，问你要什么，最后喊：

```python
scrape_cnki(keywords, max_pages=target_pages, save_mode=save_mode)
```

> **为什么用 while True**：抓完一轮问你"要不要再来一轮"，是就清屏继续，不用重开程序。

### 第 2 步：开浏览器 —— `scraper.py / scrape_cnki`

```python
with sync_playwright() as p:          # 启动 Playwright（控制浏览器的库）
    browser = p.chromium.launch(
        channel="msedge",             # 优先用系统自带的 Edge
        headless=False,               # False = 显示出来，你能看到它在搜
        args=["--start-maximized"],
    )
    context = browser.new_context(user_agent="...伪装成普通 Chrome...")
    page = context.new_page()         # page 就是那个浏览器标签页，后面全靠它
```

- `browser` = 整个浏览器；`context` = 一套独立的会话（cookie 等）；`page` = 一个标签页。
- `headless=False` 是故意的——知网要你手动过滑块，得让你看得见。

### 第 3 步：抓一个关键词 —— `_scrape_keyword`

填词、点搜索（注意这几个动作现在都带了 15 秒超时，卡住会及时报错而不是干等）：

```python
page.fill("input.search-input", keyword, timeout=15000)   # 把关键词填进搜索框
page.click("input.search-btn", timeout=15000)             # 点搜索按钮
```

> `"input.search-input"` 这种叫**选择器（selector）**，是"在网页里怎么找到这个框"的地址。

进翻页循环，把每行标题抓出来：

```python
rows = page.query_selector_all("table.result-table-list tbody tr")  # 找到结果表格所有行
for row in rows:
    title_el = row.query_selector("td.name a")   # 每行里那个标题链接
    if title_el:
        title = title_el.inner_text().strip()     # 取出标题文字
        results.append([title])                   # 存进结果列表
```

翻下一页：

```python
next_btn = page.query_selector("a#PageNext")   # 找"下一页"按钮
if next_btn:
    next_btn.click(timeout=15000)              # 点它
    time.sleep(random.uniform(4, 8))           # 随机等几秒，装得像人、别被反爬盯上
else:
    break                                       # 没有下一页 = 到底了
```

> 那些 `time.sleep(random.uniform(...))` 全是**故意放慢、随机化**，模拟人手速，降低被知网封的概率。

### 第 4 步：存盘 —— `exporter.py`

回到 `scrape_cnki` 末尾，按你选的模式存：

```python
if save_mode == "single":
    _save_single(keywords[0], all_results.get(keywords[0], []))
elif save_mode == "multi_split":
    _save_multi_split(all_results)     # 每个词一个 Excel
elif save_mode == "multi_merge":
    _save_multi_merge(all_results)     # 所有词汇总进一个 Excel 的多个 Sheet
```

文件名里加了时间戳，避免重复抓同一个词时把旧文件覆盖掉。

## 4. 两块最容易看晕的地方（重点）

### 4.1 「中途中断也不丢已抓数据」是怎么做到的

这是全项目最绕、但最重要的设计。目标：用户按 Ctrl+C、或直接关掉浏览器，**已经抓到的标题不能丢**。

靠三样东西配合：

1. **一个模块级开关 `_stop_requested`**：翻页循环里截到 Ctrl+C 时把它设成 `True`，通知上层"该停了，但已有数据要保留"。

```python
except KeyboardInterrupt:
    _stop_requested = True   # 不是直接崩，而是举手说"停"
    break
```

2. **`results` 在 try 之前就建好**：所以无论中途怎么炸，已抓的部分都还在，能被记账：

```python
results = []
try:
    results = _scrape_keyword(page, keyword, max_pages)
except PlaywrightError:
    _stop_requested = True   # 浏览器被关了
all_results[keyword] = results   # 不管成功失败，都把当前结果存下来
```

3. **`finally` 里用 `BaseException` 兜底关浏览器**：

```python
finally:
    if browser:
        try:
            browser.close()
        except BaseException:   # 注意是 BaseException 不是 Exception
            pass
```

> **为什么是 `BaseException`**：Ctrl+C（`KeyboardInterrupt`）不属于普通的 `Exception`，
> 用 `except Exception` 接不住，它会从 finally 里"逃出去"、跳过后面的存盘。
> 用 `BaseException` 才能确保关浏览器这步不会把存盘带崩。
> （审计时这行被标过"吞掉所有异常"，是已知的取舍，不是 bug。）

### 4.2 遇到知网安全验证页（滑块）怎么办 —— `_handle_verify`

判据很简单：**网址里有没有 `/verify`**。有就说明跳到验证页了。

```python
def _handle_verify(page) -> bool:
    if "/verify" not in page.url:
        return False              # 没验证，正常往下走

    window.bring_to_front()       # 把浏览器窗口拽到最前（window.py 干的）
    print_verify_alert()          # 控制台红框提醒你去过滑块

    waited = 0.0
    while "/verify" in page.url:   # 一直等到你手动过验证、网址离开 /verify
        if waited >= _VERIFY_WAIT_TIMEOUT:   # 最多等 180 秒，超时就放弃、去存盘
            return True
        if waited >= next_notice:            # 每 15 秒打一行"还在等，剩 N 秒"
            _console.print(f"...剩余约 {int(_VERIFY_WAIT_TIMEOUT - waited)} 秒...")
            next_notice += _VERIFY_NOTICE_INTERVAL
        time.sleep(1)
        waited += 1
    return True
```

> 这就是我们刚修过的地方：**上限从 300 秒降到 180 秒**，并且**每 15 秒报一次剩余时间**，
> 不再像以前那样静默卡着、让你以为死机了。

## 5. 三种存盘方式（`exporter.py`）

| 模式 | 触发 | 结果 |
|------|------|------|
| `single` | 单关键词 | 一个 Excel：`cnki_titles_<词>_<时间戳>.xlsx` |
| `multi_split` | 多关键词 + 选"分文件" | 每个词各一个 Excel |
| `multi_merge` | 多关键词 + 选"汇总" | 一个 Excel，每个词一个 Sheet（Sheet 名超 31 字会截断 + 去重） |

存不进桌面（比如没权限）时，会自动退而存到程序所在目录，不会直接报错丢数据。

## 6. 名词小抄（看代码时对照）

| 你看到的 | 大白话 |
|---------|--------|
| `Playwright` | 用代码控制浏览器的工具库 |
| `page` | 一个浏览器标签页，所有操作都对它 |
| `page.goto(url)` | 让标签页打开某网址 |
| 选择器 `"td.name a"` | "在网页里怎么定位某个元素"的地址 |
| `wait_for_selector` | 等某个元素出现（最多等 N 毫秒） |
| `timeout=15000` | 最多等 15 秒，超了就报超时 |
| `rich` / `_console` | 让控制台输出带颜色、进度条、横幅的库 |
| `Progress` | 那个翻页进度条 |
| `openpyxl` | 写 Excel 文件的库 |

## 7. 想自己改 / 跑的时候

- **改抓哪些字段**：看 `_scrape_keyword` 里的选择器（`td.name a` 等）。知网改版时，最先要改的就是这些。
- **改等待/超时**：搜 `time.sleep`（节奏）、`timeout=`（超时）、`_VERIFY_WAIT_TIMEOUT`（验证等待上限）。
- **改存盘格式**：看 `exporter.py`。
- **本地源码跑**：`pip install -r requirements.txt` → `playwright install chromium` → `python run.py`。
